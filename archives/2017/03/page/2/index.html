jQuery全屏滚动插件fullPage使用中出现的小问题总结

Jelon 发表于 2017-03-17    标签： 首屏动画 点击箭头调到下一屏 fullPage

前言

最近在做一个全屏滚动并且带有动画效果的页面,使用的是jQuery的全屏滚动插件–fullPage

在实现功能的时候,出现了一些小问题,现在记录一下解决的方法

fullPage的使用方法

引入文件

<link rel="stylesheet" href="css/jquery.fullPage.css">

<script src="js/jquery.min.js"></script>

<script src="js/jquery-ui.min.js"></script>

<script src="js/jquery.slimscroll.min.js"></script>

<script src="js/jquery.fullPage.min.js"></script>

HTML文件

<div id="fullpage" class="fullpage">
    <div id="part1" class="section">第一屏</div>
    <div id="part2" class="section">第二屏</div>
    <div id="part3" class="section">第三屏</div>
    <div id="part4" class="section">第四屏</div>
</div>

JS文件

$(function(){
    $('#fullpage').fullpage();
});

想要fullPage插件可以达到预期的效果,需要进行配置

这是我自己的配置:

$(function(){
    if( $('#fullpage').length>0){
        $('#fullpage').fullpage({
            resize: true,           //字体是否随着窗口缩放而缩放        
            verticalCentered: false,   //内容是否垂直居中
            scrollOverflow: false,     //内容过多是否显示滚动条
            menu:true,      //绑定菜单
            navigation: true,      //是否显示导航条
            anchors: ['page1', 'page2', 'page3', 'page4'],


            //滚动到某一屏后的回调函数，接收 anchorLink 和 index 两个参数。
            //anchorLink 是锚链接的名称
            //index 是section的索引，从1开始计算   
            afterLoad: function (anchorLink, index) {
                $('.section').removeClass('current');
                setTimeout(function () {
                    $('.section').eq(index -1).addClass('current');
                }, 100);
            },

            //在配置完上面之后发现,首屏的Css3动画没有效果,所以配置了下面就好了
            //这个回调函数只是在生成页面结构的时候调用,用来初始化首屏
            afterRender: function(){
                $('#part1').addClass('current')
            }

        })

    }
})

每一屏中点击按钮跳到下一屏

只需要在每一屏设置一a链接,href属性的值是下一屏的#id值

<div id="part1" class="section">第一屏
    <a href="#part2">跳到下一屏</a>
</div>

<div id="part2" class="section">第三屏
    <a href="#part3">跳到下一屏</a>
</div>

每屏的导航器设置不同的颜色

在设置之前,导航器是有不同的颜色的,如下图:

现在设计图的要求是更改,如下图,并且第一屏和第三屏颜色一样,第二屏和第四屏颜色一样:

配置每个圆点的大小

#fp-nav span,
.fp-slidesNavspan{width: 16px; height: 16px;}

JS文件

那么我们如果修改fullpage.css中的样式,那么每屏的颜色还是一样,所以我用JS来控制了他的颜色

$('#fullpage').fullpage({
            resize: true,
            verticalCentered: false,
            scrollOverflow: false,
            menu:true,
            navigation: true,
            anchors: ['page1', 'page2', 'page3', 'page4'],
            afterLoad: function (anchorLink, index) {
                $('.section').removeClass('current');
                setTimeout(function () {
                    $('.section').eq(index -1).addClass('current');
                }, 0);
                //如果是第一屏和第三瓶配置一种颜色
                if(index == 1 || index == 3){
                    $('#fp-nav span,.fp-slidesNavspan').css('background','#87c2f8');
                    $('#fp-nav li .active span,.fp-slidesNav.active span').css('background','#fff');
                    $('#fp-nav li a').on('mouseenter',function(){
                        $(this).children().css('background','#fff');
                    })
                    $('#fp-nav li a').on('mouseleave',function(){
                        $(this).children().css('background','#87c2f8');
                        $('#fp-nav li .active span').css('background','#fff');
                    })
                    $('#fp-nav span,.fp-slidesNavspan').css('+background','url(../images/y1.jpg)');
                    $('#fp-nav li .active span,.fp-slidesNav.active span').css('+background','url(../images/y2.jpg)');
                //如果是第二屏和第四瓶配置一种颜色
                }else if(index == 2 || index ==4){
                    $('#fp-nav span,.fp-slidesNavspan').css('background','#bfdffc');
                    $('#fp-nav li .active span,.fp-slidesNav.active span').css('background','#47a2f5');
                    $('#fp-nav li a').on('mouseenter',function(){
                        $(this).children().css('background','#47a2f5');
                    })
                    $('#fp-nav li a').on('mouseleave',function(){
                        $(this).children().css('background','#bfdffc');
                        $('#fp-nav li .active span').css('background','#47a2f5');
                    })
                }
            },
            afterRender: function(){
                $('#part1').addClass('current')
            }
        });

这样就配置好了,真的好麻烦,但是我还没有找到简单的方法

从上到下是交叉轴

wxml结构：

  <view class="item">

  </view>
  <view class="item itemText">
      2
  </view>
  <view class="item">
      3
  </view>

  </view>

wxss样式：

.container{
    height: 100%;
    width: 100%;
    background-color: lightyellow;
    padding-top:50rpx;
}
.item {
    width: 100rpx;
    height: 100rpx;
    background-color:pink;
    text-align:center;
    line-height:100rpx;
    margin:10rpx;
}

那么他的默认的item的显示方式是：

用flex布局,只需要给他的父容器类名为container,加上display:-webkit-box;的样式
再加上这个样式之后,默认是按照排列

flex容器的属性(给容器添加)

-webkit-box-orient

决定元素的排列方向,也就是决定是主轴还是交叉轴

-webkit-box-orient:horizontal; 默认方向,水平,主轴方向水平方向


-webkit-box-orient:vertical;竖着排列主轴方向是竖着的

-webkit-box-direction

决定元素如何换行,当元素超过一行,决定他如何换行

我们多添加几个元素,现在元素共有10个,他的排列方式
每一个item的大小是变形的

我们可以在一行排列不下时,设置他的排列方式
flex-wrap:nowrap;  不换行


flex-wrap:wrap;  换行


-webkit-box-direction:reverse;  反转

flex-flow

是flex-direction和flex-wrap的简写

flex-flow: row wrap; 代表每一个item水平排列并且元素超过一行,换行

justify-content

元素在主轴上的对齐方式

justify-content:flex-start;默认值,从左边开始进行对齐


justify-content:flex-end;从右边开始进行对齐


justify-content:center;  元素在主轴上居中对齐


justify-content:space-around;元素两边的距离是一样的


justify-content:space-between;两边对齐,中间的元素间隔是一样的

align-items

元素在交叉轴的对齐方式

align-items:flex-start;在交叉轴上的头部


align-items:flex-end;在交叉轴上的尾部


align-items:center;在交叉轴上的中间


align-items:stretch;

在item设置高度时候,显示方式跟   align-items:flex-start一样

当每一个item元素不设置的高度的时候,高度会占满整个容易


align-items:baseline;让元素以元素中的文字的基线对齐

给第二个元素设置样式.itemText {line-height:150rpx;}


然后给父容器设置align-items:baseline;样式

flex元素的属性(无法用在容器,只用于元素中)

flex-grow (相当于栅格系统,默认值是0)

当容器有多余空间时,元素的放大比例

flex-grow:0; 默认值是0,当容器有多余的空间,元素不放大

flex-grow:1;当容器有多余的空间,容器放大1倍

现设置每一个 .item{flex-grow:1;},第二个元素: .itemText {flex-grow:3;}

flex-shrink(默认值是1,不缩小)

当容器空间不足时,元素的缩小比例

flex-shrink:1;当空间不足时,默认等比缩小

.itemText {flex-shrink:10;}当空间不足时,等比缩小10倍

flex-basis

flex-basis的单位不能用rpx

.itemText {flex-basis:200px;}

flex

是grow、shrink、basis三个的简写

.itemText {flex: 1 1.5 200px;}

第一个1代表flex-grow:1;当容器有多余的空间,容器放大1倍
第二个1.5代表flex-shrink:1;当空间不足时,默认等比缩小,
第三个200px代表flex-basis:200px; 元素在主轴上占据的空间是200px

order

定义元素的排列顺序

我们给 .item{order:4;}
再给 .itemText {order:3;}

align-self

定义元素自身的对齐方式
这个属性可以覆盖掉 justify-content、align-items的效果

align-self:flex-start;在交叉轴上的头部

align-self:flex-end;在交叉轴上的尾部

align-self:flex-center;在交叉轴上的中间

align-self:stretch;
在item设置高度时候,显示方式跟   align-items:flex-start一样
当每一个item元素不设置的高度的时候,高度会占满整个容易


align-self:baseline;让元素以元素中的文字的基线对齐

微信小程序开发基本配置和使用

Jelon 发表于 2017-03-14    标签：

微信小程序的文件

在微信小程序中,我们把所有的页面都放在一个统一的文件夹下,以单个文件夹存在,每个页面下又分别由三个页面组成,分别是 js、wxml、wxss

js、wxml、wxss 这三个文件就构成了一个完整的页面

三个文件的作用

js ：定义页面的逻辑

调用了一个page方法，是一个全局函数，用来创建页面对象

onLoad方法页面加载时触发

wxml ：定义页面的结构 语法是XML语法(Wei Xin Markup Language)

wxss ：定义页面的样式 遵循是css语法(Wei Xin Style Sheet)

index.json的配置

在页面的json文件主要是为了覆盖app.json中window 配置项的内容

wxml标签

view相当于html中的div

text标签定义文本

image图片标签

block专门在做一些循环的时候,又不想在循环的时候在外层新加一些标签,block在循环完成后不会显示

app.json

在app.json文件中定义了一个pages的属性,这个属性定义了我们的应用程序可用的所有的页面(所以pages中的页面必须与app.json建立关联)

"pages":[
    "pages/index/index",  //代表初始页面
    "pages/logs/logs"
  ]

app.js

有一个全局函数是 App,作用是用来指定生命周期

App({
    onLaunch(){  //在App这个对象中定义了一个onLaunch方法
        console.log('msg')    
    },
    foo(){   //定义一个额外方法
        return 'bar'
    }
})

在页面的js文件中 引入 const app=getApp(),那么在页面中就可以调用app.js中定义的方法了

var app = getApp()

Page({
    onLoad(){
        app.foo();  
    }
})

app.wsxx

整个程序共享的样式(公共样式)

数据加载到页面

小程序提供了一个属性:page,这个属性可以是一个对象:这个对象中的所有的成员都可以在界面上直接被访问到

在index.js中

Page({
    data:{
        message:'hello world'
    },
    onLoad(){
        //页面加载就把数据呈现在页面
        this.setData({
            message:Date.now() //获取当前时间
        })  
    }
})

在页面上显示时

在index.wxml中
<text>{{message}}<text>

访问网络资源

微信开发者工具在更新到最新版的时候,发现请求时会报错

错误信息:GET https://api.douban.com/v2/movie/in_theaters 400 (Bad Request)          asdebug.js:1 

百度了一下解决方案 : 

 header: {
    // 'content-type': 'application/json'
    'Content-Type': 'json'
}

微信官方提供了API,不存在跨域的问题

wx.request({
  url: 'test.php', //仅为示例，并非真实的接口地址
  data: {
     x: '' ,
     y: ''
  },
  header: {
      'content-type': 'application/json'
  },
  success: function(res) {
    console.log(res.data)
  }
})

把访问外部链接获取的数据放到list当中

Page({
    data:{
        message:'hello world!',
        list:[]
    },
    onLoad(){
        const _this = this
        //页面加载时候执行
        wx.request({
          url: 'https://api.douban.com/v2/movie/in_theaters',
          data: {},
          header: {
              'Content-Type':'json'
          },
          success: function(res) {
              console.log(res.data)
            _this.setData({
                list: res.data.subjects
            })
          }
        })
    }
})

在页面上显示的时候

<view>
  <view class="item" wx:for="{{ list }}">
    <image src="{{ item.images.small }}"></image>
    <text>{{ item.title }}</text>
  </view>
</view>

tabBar 是一个数组，只能配置最少2个、最多5个 tab，tab 按数组的顺序排序。

{
  "pages": [
    "pages/index/index",

    }, {
      "pagePath": "pages/logs/logs",
      "text": "日志"
    }]
  },
  "networkTimeout": {
    "request": 10000,
    "downloadFile": 10000
  },
  "debug": true
}

微信小程序模板的使用

Jelon 发表于 2017-03-13    标签：

前言

我们在应用程序开发的过程当中,很说时候会有一些共用的代码,我们不可能说每次使用的时候,都去复制粘贴一下

这就需要模板来方便我们的开发

模板的建立

你需要在Pages文件夹下新建一个名为template的文件夹作为模板的目录

这是第一种方式:

在template文件夹下,新建一个header.wxml文件,文件内容是
<view>这是头部</view>

然后在你使用到模板的页面下引入<include src="../template/header"/>

第二种方式我们在template文件夹下新建一个footer.wxml文件,文件内容为:

<template name="footer">这是底部内容</template>

使用<import src="../template/footer"/>方式导入要使用的文件
在这里可能我们会定义了很多的模板,所以在引入时候我们要声明使用的是哪个模板
<template is="footer"> 这个is属性跟template的name属性一致

我们也可以给第二种方式动态设置数据

<template name="footer">这是底部内容 {{text}} </template>

<template is="footer" data=" {{ text:"动态的数据" }}">

swiper

滑块视图容器。

属性名                      类型                   默认值                   说明

indicator-dots              Boolean               false                   是否显示面板指示点
indicator-color              Color                   rgba(0, 0, 0, .3)       指示点颜色（这个属性目前暂未启用）
indicator-active-color      Color                   #000000                   当前选中的指示点颜色 （这个属性目前暂未启用）
autoplay                  Boolean               false                   是否自动切换
current                      Number               0                       当前所在页面的 index
interval                  Number               5000                       自动切换时间间隔
duration                  Number               500                       滑动动画时长
circular                  Boolean               false                   是否采用衔接滑动
bindchange                  EventHandle                            current 改变时会触发 change 事件，event.detail = {current: current}

swiper-item
仅可放置在组件中，宽高自动设置为100%。

示例代码：

<swiper indicator-dots="{{indicatorDots}}"
  autoplay="{{autoplay}}" interval="{{interval}}" duration="{{duration}}">
  <block wx:for="{{imgUrls}}">
    <swiper-item>
      <image src="{{item}}" class="slide-image" width="355" height="150"/>
    </swiper-item>
  </block>
</swiper>
<button bindtap="changeIndicatorDots"> indicator-dots </button>
<button bindtap="changeAutoplay"> autoplay </button>
<slider bindchange="intervalChange" show-value min="500" max="2000"/> interval
<slider bindchange="durationChange" show-value min="1000" max="10000"/> duration

示例代码属性解读:

indicator-dots : 轮播图的点盒子,可以直接设置为true
autoplay:是否自动播放,也可以把值设为 true
interval:自动播放的时间间隔,也可以把时间写死,如:8000(单位是毫秒)
duration:每次轮播的速度,可以固定,默认1000,代表1s

js中的配置

Page({
  data: {
    imgUrls: [
      'http://img02.tooopen.com/images/20150928/tooopen_sy_143912755726.jpg',
      'http://img06.tooopen.com/images/20160818/tooopen_sy_175866434296.jpg',
      'http://img06.tooopen.com/images/20160818/tooopen_sy_175833047715.jpg'
    ],
    indicatorDots: false,    //可以在wxml文件中定义
    autoplay: false,
    interval: 5000,
    duration: 1000
  },
  changeIndicatorDots: function(e) {
    this.setData({
      indicatorDots: !this.data.indicatorDots
    })
  },
  changeAutoplay: function(e) {
    this.setData({
      autoplay: !this.data.autoplay
    })
  },
  intervalChange: function(e) {
    this.setData({
      interval: e.detail.value
    })
  },
  durationChange: function(e) {
    this.setData({
      duration: e.detail.value
    })
  }
})

微信小程序事件

Jelon 发表于 2017-03-13    标签：

微信小程序事件

什么是事件

事件是用户的一种行为

事件的分类

事件分为冒泡事件和非冒泡事件：

冒泡事件：当一个组件上的事件被触发后，该事件会向父节点传递。
非冒泡事件：当一个组件上的事件被触发后，该事件不会向父节点传递。

   类型                      触发条件
   touchstart              手指触摸动作开始
touchmove              手指触摸后移动
touchcancel              手指触摸动作被打断，如来电提醒，弹窗
touchend              手指触摸动作结束
tap                      手指触摸后马上离开
longtap                  手指触摸后，超过350ms再离开

非冒泡事件：除上表之外的其他组件自定义事件如无特殊申明都是非冒泡事件，如

的submit事件，input标签的input事件，的scroll事件，(详见各个组件)

bind和catch的分别

catch绑定的事件会阻止冒泡

bind绑定事件会进行冒泡

事件对象

currentTarget 绑定的这个组件

target 触发事件源控件

dataset 方便给view添加上我们自己的自定义属性 data-title=”hello”
